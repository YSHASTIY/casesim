# CRATE — симулятор кейсов CS2 с реальными ценами Market.CSGO

Статический сайт (HTML/CSS/JS, без бэкенда). В кейсах — реальные предметы CS2
с картинками и живыми ценами в рублях с [market.csgo.com](https://market.csgo.com).
Клик по предмету открывает Market.CSGO с фильтром по этому скину (все износы:
прямо с завода, немного поношенное, после полевых испытаний, поношенное,
закалённое в боях, а также StatTrak™ / Souvenir).

## Откуда берутся цены

Прямой запрос `market.csgo.com/api/v2/prices/RUB.json` из браузера блокируется
CORS. Поэтому цены парсятся скриптом и кладутся в статический файл:

- **`scripts/generate_data.py`** — парсит цены (market.csgo.com) и базу предметов
  [ByMykel/CSGO-API](https://github.com/ByMykel/CSGO-API) (картинки, редкость,
  износы), собирает `data/items.json`. Только стандартная библиотека Python.
- **`data/items.json`** — сгенерированные данные: 5 кейсов с реальными предметами,
  ценами, картинками и ссылками на маркет. Сайт грузит его как обычный файл.
- **`.github/workflows/update-prices.yml`** — GitHub Action раз в сутки запускает
  генератор и коммитит свежий `data/items.json`. Бесплатно.

Цены обновляются раз в сутки (не real-time). Время последнего обновления
показывается на страницах кейсов («Цены обновлены: …»).

## Цена кейса

`price = EV × 1.18`, где `EV` — ожидаемая стоимость выпадения при взвешенном
ролле по редкости (Mil-Spec 79.92%, Restricted 15.98%, Classified 3.2%,
Covert 0.64%, ножи/перчатки 0.26%). Перевес казино ~15%: большинство спинов
проигрышные (дешёвые предметы дешевле кейса), редкие ножи дают крупный выигрыш.

## Локальный запуск

Открыть `index.html` через любой статический сервер (нужен для fetch `data/`):

```bash
python3 -m http.server 8000
# http://localhost:8000
```

Регенерировать данные вручную:

```bash
python3 scripts/generate_data.py
```

## Админ-панель, контроль выигрышей и сервисный слой (обновление)

Сайт остаётся **статическим** (без бэкенда). Админка, CRUD каталога и контроль
выигрышей реализованы как **клиентский сервисный слой** (`js/services/*`) с
хранением в `localStorage` и парольным входом. Вся логика цены/дропа вынесена в
сервисный слой и переиспользуется сайтом и тестами.

### Вход в админку

1. Откройте `admin.html` (ссылка «Админ» в футере главной).
2. Пароль по умолчанию: **`crate-admin-2026`** — **обязательно смените** перед
   деплоем. Хранится только SHA-256-хэш (`js/admin-config.js`), не plaintext.

Сменить пароль без раскрытия plaintext:

```bash
node -e "console.log(require('crypto').createHash('sha256').update('НОВЫЙ_ПАРОЛЬ').digest('hex'))"
```

Полученный хэш вставьте в `ADMIN_PASSWORD_HASH` (`js/admin-config.js`) **или**
задайте разово в консоли браузера:
`localStorage.setItem('crate-admin-hash', '<хэш>')`. См. также `.env.example`.

> **Безопасность.** Клиентская проверка пароля **не защищена от подделки** —
> любой сервисный слой в браузере можно обойти через DevTools. Для боевого
> использования вынесите проверку и контроль выигрышей на сервер: сервисные
> модули написаны как UMD (браузер + Node `require`) и переносятся на
> Node/Express без изменений.

### Возможности админки

- **Дашборд** — кол-во кейсов/скинов, открытий сегодня, баланс, средний RTP,
  bar-chart RTP по кейсам.
- **Кейсы (CRUD)** — название, иконка, цвет, описание, статус публикации, состав.
  Цена **автосчитается** по существующей формуле `price = round(EV·1.18/10)·10`
  и пересчитывается при изменении состава; есть тумблер **ручной цены**.
  Валидация весов (> 0) и показ **нормализованных %** по каждому скину.
  Кнопка **«Авто-набор»** — сбалансированный случайный состав из пула.
- **Скины** — таблица всего каталога с поиском.
- **Контроль выигрышей** — режимы `off | manual | auto`, % игроков «в плюс»,
  cap = стартовый баланс × множитель / + фикс. сумма, дневной потолок.
- **Аудит** — журнал решений контроля (естественный дроп → выданный, причина, cap).
- **Импорт/Экспорт** — импорт скинов JSON, экспорт/импорт всей конфигурации, сброс.

### Контроль выигрышей (Phase 3)

Реализовано как реальные настройки + логика (`js/services/profit.js`,
`js/services/drop.js`), **без скрытого RNG-хака**:

- **auto** — игрок детерминированно (seed = `userId + дата`) попадает в группу
  «можно/нельзя в плюс» на день (не перекатывается на каждый кейс).
- В момент дропа честный взвешенный ролл сверяется с дневным cap игрока. Если
  выпавший предмет превышает cap — переролл среди доступных предметов; если все
  превышают — выдаётся **самый дешёвый**. Каждое капнутое решение пишется в аудит.
- **manual** — админ помечает конкретных игроков allow/disallow.
- Cap для «нельзя в плюс» = стартовый баланс дня (пример: сделал себе 50 000 ₽ —
  выше ~50 000 не поднимется; при ratio 1.2 «можно»-группа растёт до 60 000 ₽).

### Полный каталог предметов CS2 (Phase 2)

`python3 scripts/generate_data.py` дополнительно пишет **`data/skins-pool.json`**
и **`data/skins-pool.js`** — плоский массив **ВСЕХ** предметов CS2 (в текущем
прогоне ~14 400): оружие, ножи, перчатки, **наклейки, брелоки, нашивки, значки
(пины) и сами кейсы/капсулы**. Источник — публичный датасет
[ByMykel/CSGO-API](https://github.com/ByMykel/CSGO-API) (эндпоинты
`skins`, `stickers`, `keychains`, `patches`, `collectibles`, `crates`);
цены сопоставляются с `market.csgo.com` по `market_hash_name` (где есть).

Каталог **не копируется в `localStorage`** (он ~10 МБ) — `admin.html` подключает
`data/skins-pool.js` тегом `<script>`, а `Catalog.allSkins()` читает
`window.CRATE_SKIN_POOL` напрямую. В `localStorage` попадают только скины,
импортированные вручную во вкладке «Импорт/Экспорт».

Формат объекта каталога:

```json
{ "name": "AK-47 | Redline", "category": "weapon", "weapon": "AK-47",
  "tier": "classified", "statTrak": false, "price": 1200,
  "image": "https://…", "wear": "Field-Tested",
  "rarityName": "Classified", "rarityColor": "#d32ce6", "marketUrl": "https://…" }
```

Обязательны `name`, `tier`, `price`; остальное опционально. `tier` ∈
`milspec | restricted | classified | covert | rare`. `category` ∈
`weapon | knife | gloves | sticker | charm | patch | pin | case`. Для оружия и
ножей со StatTrak генерируется **отдельный вариант** (`statTrak: true`,
имя `StatTrak™ …`) — как на маркете.

**Пересобрать каталог:** `python3 scripts/generate_data.py` (нужен интернет для
доступа к ByMykel/CSGO-API и market.csgo.com; недоступные источники пропускаются).

### Фильтры при сборке кейса (как на маркете CS:GO)

В окне «Добавить предмет в кейс» (вкладка «Кейсы» → редактирование) — панель
фильтров с чекбоксами: **категория**, **редкость**, **StatTrak да/нет** и
**оружие (базовый предмет)**. Чекбоксы внутри одной группы объединяются по «ИЛИ»,
группы — по «И», и всё вместе — со свободным поиском по названию. Пример: ввести
`apple`, отметить оружие `AK-47`, редкость `Covert`, StatTrak `Нет` — все условия
одновременно. Кейсы также присутствуют в списке, поэтому можно собрать кейс,
содержащий другие кейсы (выданный кейс просто кладётся в инвентарь — рекурсии при
открытии не происходит).
